This extension generates Bambda code snippets from plain-language descriptions using Burp AI. Select the Bambda type that matches your intended use, describe the desired behavior, and the extension returns Java code ready to paste into a Bambda editor.

Features

  • Generates Bambdas for table filters (Proxy HTTP history, WebSockets history, Site map, Logger), custom columns, Repeater custom actions, match-and-replace rules, and custom scan checks (passive and active).
  • Context-aware prompts supply the AI with the correct input variable, its type, and the required return value for each Bambda type, ensuring the output targets the appropriate API.
  • Adjustable temperature spinner (0.0-2.0, default 0.2) controls how deterministic or varied the generated output is.
  • Generated code is displayed in an editable text editor so it can be reviewed and adjusted before use.
  • One-click copy to clipboard for the generated snippet.
  • Per-type memory retains your description and last result separately for each Bambda type within a session, so switching between types does not discard previous work.
  • AI response is treated as untrusted text and displayed only in a plain-text editor; any surrounding Markdown code fences are stripped before display.

Usage

  1. Open the Bambda Generator tab.
  2. Select the Bambda type from the dropdown that matches where you intend to use the snippet (for example, Proxy HTTP history table filter or Repeater custom action).
  3. Enter a plain-language description of what the Bambda should do in the description field (for example, show only in-scope requests that returned a 500 status).
  4. Optionally adjust the Temperature value to control output variation.
  5. Click Generate and wait for the AI response to appear in the output editor.
  6. Review and edit the generated code as needed, then click Copy output.
  7. Paste the snippet into the appropriate Bambda editor and test it against your traffic before relying on it.
